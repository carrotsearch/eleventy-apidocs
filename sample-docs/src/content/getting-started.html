Getting started

Overview

@carrotsearch/eleventy-apidocs is an Eleventy plugin that turns a tree of authored HTML files into a navigable documentation site with shared layout, navigation, theme switching, code highlighting, responsive images, and a per-page table of contents.

It assumes you write content in plain HTML rather than Markdown. The authoring surface is small — a handful of conventions on top of standard tags — so the source stays close to what eventually ships in the browser.

Install

Add Eleventy and the theme to your project:

pnpm add -D @11ty/eleventy
pnpm add @carrotsearch/eleventy-apidocs

Requirements

Node 20.11 or later is required. The plugin is published as ESM and expects an ESM consumer config.

Workspaces

Inside a pnpm workspace, you can reference the theme with workspace:* while it lives in the same repo as your docs.

Configure Eleventy

In eleventy.config.js, register the plugin and point it at your content directory: 

import apidocs from "@carrotsearch/eleventy-apidocs";

export default async function (eleventyConfig) {
  return apidocs(eleventyConfig, {
    contentDir: "src/content"
  });
}

Options

The plugin accepts the following options. Each one is optional — paths default to conventional locations under src/.

navigation
Path to the navigation manifest, either a flat array of articles or a chaptered shape. Default: src/navigation.json.
logo
Path to an HTML fragment rendered in the header's logo slot. Default: src/logo.html.
footer
Path to an HTML fragment rendered in the page footer. Default: src/footer.html.
contentDir
Directory of article HTML files, relative to the project root. Eleventy's input directory is set from this. Default: src/content.
variables
Map of $NAME$ placeholders to replacement values. The pipeline substitutes them into every emitted page at build time. Default: {}.

Custom pipeline passes

Pass transformers and finalizers to hook into the HTML pipeline before and after the built-in passes. Each receives a Cheerio instance to mutate in place.

Authoring contract

Each article is a single HTML file. The expected outer shape is an <article> with an <h1> title and one or more <section> blocks, each with a stable id: 

<article>
  <h1>Title</h1>
  <section id="intro">
    <h2>Intro</h2>
    <p>…</p>
  </section>
</article>

Section ids anchor the per-page table of contents and act as stable deep links. Nested sections appear as nested ToC entries.
